    Notes

  • Debugg throughout your code and test frequently to make sure your code is performing like you want
  • You can debug with the debugger keyword but it's cleaner to use breakpoints throughout your code
  • TDD- Test Driven Development. Writing tests before code, once code is written, its continually tested so it can be refactored
  • Testing frameworks provide structure for more productive tests
  • Try and catch blocks are important when handling errors and exceptions. If the try part fails then the catch part catches it and handles the error
